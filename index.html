Comandos Git

Comando Git Descripción Parametros Ejemplo
git init Inicializa un nuevo repositorio Git en el directorio actual. Ninguno git init
git clone <URL> Clona un repositorio remoto en el directorio local. [url] Enlace del repositorio remoto que será clonado. git clone https://github.com/ejemplo/repo.git
git add <archivo> Agrega un archivo o cambios al área de preparación. . | nombre del archivo para preparar git add . | git add matematicas.html
git commit -m "Mensaje" Realiza un commit de los cambios en el área de preparación. Mensaje del commit git commit -m "Nueva funcion añadida"
git status Mostrar los archivos modificados en el directorio de trabajo, peparados para un proximo commit Ninguno git status
git reset [fichero] Desenvolver un archivo conservando los cambios en el directorio de trabajo [fichero] indica el fichero que sera descartado git reset archivoadescartar.txt
git reset -- soft HEAD^ Desenvolver el ultimo commit conservando los cambios en el directorio local Ninguno git reset -- soft HEAD^
git diff Diferencia de lo que se cambia pero no se pone en escena Ninguno git diff
git diff --staged Diferencia de lo que se ha puesto en escena pero aun no se ha hecho commit Ninguno git diff --staged
git branch lista de las ramas y aparecera un * al lado de la rama actualmente activa Ninguno git branch
git branch [nombre de la rama] Crear una nueva rama en el commit actual [nombre de la rama] crear rama con el nombre deseado git branch medina
git checkout -b [nombre de la rama] crea y desplazamiento a la rama indicada [nombre de la rama] aqui pondremos el nombre deseado para la rama git checkout -b medina
git branch -d [nombre de la rama] Elimina la rama indicada [nombre de la rama] aqui pondremos el nombre deseado para la rama git branch -d medina
git merge [rama] Fusionar el historial de la rama especificada en la actual [rama] nombre de la rama para fusionar en la rama actual git merge medina
git log Mostrar el historial de commits de la rama actualmente activa Ninguno git log
git log ramaB...ramaA Mostrar los commits de la ramaA que no estan en la ramaB [ramaB]nombre de la rama a no mostrar los commits [ramaA] rama a mostrar los commits git log medina...soto
git rm [fichero] Elimina el archivo del proyecto y prepara la eliminacion para el commit [fichero] nombre del fichero a eliminar git rm archivoaeliminar.txt
git mv [ruta existente] [nueva ruta] Renombra un archivo o directorio o cambiar la ruta de un archivo existente y realiza el movimiento Ruta del archivo o directorio existente y nueva ruta o nombre git mv archivo_viejo.txt archivo_nuevo.txt
git remote add [alias] [url] Añadir una URL de git como alias Nombre del [alias] y [URL] del repositorio git remote add origin https://github.com/ejemplo/repo.git
git fetch [alias] Obtener todas las ramas del git remoto nombre del [alias] git fetch origin
git push -all Transmitir los commits de la rama local a la rama del repositorio remoto [all] indica todos los commits de la rama local git push -all
git pull [alias] [rama] Obtener y fusionar los commits de la remota a la local nombre del alias y rama local git pull origin master
git tag Visualizar los tags Ninguno git tag

Conclusion

Los comandos de git son herramientas importantes para trabajar de una manera efectiva con sistemas de contro de versiones .
En la tabla anterios se coloracon comandos clave que se utilizan comunmente en el flujo de trabajo Git.
Los comandos son fundamentales para el uso de Git y la colaboracion en proyectos de desarrollo de software.
